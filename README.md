# このサイトはほぼAIで作成した自己満アプリです。
## つまるところ現在の脳腐れZ世代を体感するのと同時に、AIの進化を感じるためのサイトでもあります・・・

一応なぜこの自己満アプリを作成したか、それは私の好みのタスクやポモドーロアプリがなかったためです。
そこで、好奇心ではありますがこのアプリを制作（AIがですが笑）しました。


# CHANGELOG
All notable changes to SCHED will be documented in this file.

---

## [v1.0.6] - 2026-03-07
### Added
- オンライン内訳ポップアップをスマホ（onlineBadgeMobile）にも対応

### Fixed
- `::before` 矢印のCSS `transform` が2行重複していた問題を修正（矢印位置ズレ）
- モバイルポップアップの位置を右端に寄せ、画面外にはみ出ないよう調整
- ログイン確定前に presence が書き込まれると uid なし（ゲスト扱い）になる問題を修正
  （`_onAuthChange` 時に presence を uid 付きで再書き込みするよう変更）

---

## [v1.0.5] - 2026-03-07
### Added
- オンラインバッジをクリックするとログイン済み・ゲストの内訳ポップアップを表示
  - presenceデータに uid フィールドを追加（ログイン済みユーザーのみ）
  - バッジクリックで表示、外クリックで閉じる

---

## [v1.0.4] - 2026-03-07
### Fixed
- ログインポップアップがサイト訪問のたびに毎回表示されるバグを修正（根本原因）
  - 時計の日付変わり検知ブロック内で `S.streak.lastPopDate=''` をリセットしていたが、
    `_lastDateStr` の初期値が `''` のためページロード時に必ずこのブロックが実行され、
    毎回 `lastPopDate` がリセットされポップアップが表示されていた

---

## [v1.0.3] - 2026-03-07
### Fixed
- ストリークが途切れた日の翌日以降もログインポップアップが毎回表示されるバグを修正
  - `streakBreakPop` 表示後に `lastPopDate` を保存せず `return` していた問題を修正

---

## [v1.0.2] - 2026-03-07
### Changed
- Discord方式のpresenceへの全面置き換え提案を検討・現行設計を維持
  - Primaryタブ制御 + BroadcastChannel による「同一ブラウザ = 1人」の設計はこのアプリに適しているため変更なし

---

## [v1.0.1] - 2026-03-07
### Fixed
- presence人数カウントの `Math.max(1, ...)` を2箇所削除
  - `onValue` コールバック内（0人でも1人表示になるバグ）
  - `_setOnlineCount` 内（正確な値が渡っても最低1に上書きされていたバグ）
- `_writePresence` のエラーを握りつぶさず `console.error` で可視化

---
